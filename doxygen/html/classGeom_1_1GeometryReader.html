Detailed Description

read geometry file and generate GeometryData object must be in sequential mode, should derived from App::Reader, for the input manifest of multiple geometry files list, parallel reading is possible.

Aupported geometry file formats

  • step/stp AP214(with material and color meta data) OpenCASCADE XCAF
  • IGES/igs: OpenCASCADE XCAF reader
  • FreeCAD native format *.FCStd
  • parallel preproessor output: *.brep + *_metadata.json(*.metadata.json)

  • manifest.json textual format: a list of filename-materail json objects this json file must ended with "manifest.json", here is an example:

    [{
    "material": "one_material",
    "filename": "absolute_path or path_relative_to_this_json_file/something.stp"
    },
    {
    "material": "another material",
    "filename": "absolute_path or path_relative_to_this_json_file/something.stp"
    }]

    Note: 1. filename key can be any string contains substring "filename" such as "stp_filename" but recommend to use just "filename".

    1. currently all input geometry format must be of the same format.
