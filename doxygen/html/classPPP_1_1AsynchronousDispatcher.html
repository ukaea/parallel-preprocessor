Detailed Description

Parallel access item pairs in asynchronous without waiting other workers to complete.

For asynchronous operation, each worker can unlock items it completed; there is no needs to wait for other workers, just lock item from myRemainedItems and carry on work; the data structure myRemainedItems is protected by mutex.

single producer (myRemainedItems) and multiple consumers queue pattern (by mutex locking), There is no concept of scheduler which manages the queue, but any worker found the queue is empty will produce (fill the queue).

NOTE: myRemainedItems.size()==0 does not means all items have been processed completely, some may be locked by other workers, wait() all workers are still needed by the task_group
